## 根本问题

AI 工具没有持久记忆，人每次要手动维护上下文，累且容易断。

（你的临时解法——每次让Claude总结memory文档——正是主脑想要自动化的那件事）

---

## 核心框架（4层）

```
宪法层  → 最高原则：记什么、不记什么、输出格式，极少改动
规则层  → 执行规范：告诉AI怎么行动（CLAUDE.md / _rules.md）
执行层  → 19个Skills + 脚本：实际干活的
数据层  → 存储：inbox/ tasks/ projects/ journal/
```

### 一句话记忆口诀
> 原则管格式，规则管行为，技能来干活，数据来存档

---

## 一个想法的完整流转

> 你说："记一下，研究一下向量数据库"

```
规则层(auto-capture) 拦截
    ↓
执行层(task-capture) 解析意图、判断粒度
    ↓
数据层(inbox/ 或 tasks/) 写入文件
    ↓
后置链自动触发：更新索引 → 重建仪表盘 → git提交
```

**关键**：你不用手动维护，后置链保证系统自动保持一致。

---

## 最有价值的设计：后置链

任何变动发生后，自动串联执行：

```
文件变更 → sync_index.py → gen_dashboard.py → git push → 刷新服务
```

这是它比"手动总结memory文档"最大的进化——**变成了自动的**。

---

## 后台Agent提案系统（异步决策）

当你需要AI去研究某个问题时：

```
queue.md 写入任务
    ↓
watchdog 监听到变化
    ↓
background_agent 启动隔离Session
    ↓
Agent 研究 → 写提案(proposals/001a/001b)
    ↓
Dashboard 展示多选一
    ↓
你审批 → proposal-apply 执行
```

**核心思路**：AI异步研究，人做最终决策，不打断当前工作流。

## 关键细节

- **多工具兼容**：同一套能力，Claude Code / OpenCode / Cursor / Copilot 都能用，靠各自的配置文件适配
- **宪法不可违背**：`_constitution.md` 是最高层，所有规则不能和它冲突
- **`_index.md` 是神经中枢**：存当前焦点+优先级，每次新会话先读它，相当于"工作记忆"
- **提案多选一**：Agent不直接执行，而是给你A/B/C方案选，你决策后才应用

---

## 可迁移的模式

> **任何"AI辅助个人系统"都需要解决4件事：**
> 1. **原则**：什么值得记，格式是什么
> 2. **行为规范**：AI怎么响应不同输入
> 3. **执行能力**：谁来干具体的活
> 4. **存储**：数据放哪、怎么流转
>
> 主脑的后置链思路可复用：**任何变动都触发一条自动化串联**，保证状态一致，人不用手动维护。

## 用 Claude Code 能复刻吗？

| 能力 | 主脑(OpenCode) | Claude Code |
|------|---------------|-------------|
| 持久记忆 | ✅ | ✅ 靠 `_index.md` |
| 后置链自动触发 | ✅ | ✅ 靠 `CLAUDE.md` |
| 后台 Agent 异步跑 | ✅ API控制 | ❌ 手动触发 |
| 定时任务 | ✅ | ⚠️ 要配 Windows 计划任务 |

**核心差距只有一个**：后台 Agent 异步执行——CC 不在你面前就不动。

**用 API 方式为什么能解决**：OpenCode 是常驻服务，程序可以随时调用，不需要人在场。如果部署到云服务器，即使关机也能跑。

**CC 用户的最小可行方案**：
- `_index.md` 放在 GitHub（跨设备同步）
- 全局 `CLAUDE.md` 写启动/结束规则
- 对话结束说"先到这" → AI 自动更新 `_index.md`

## 和 echo-scheduler 的关系

| | echo-scheduler | 主脑 MasterMind |
|--|--|--|
| 解决什么 | 多AI会话协调 | 个人工作记忆持久化 |
| 核心机制 | 消息总线+调度 | 四层架构+后置链 |
| 用户角色 | 监控者 | 决策者 |

两者可以组合：主脑管理任务和记忆，echo-scheduler 负责跨会话协调执行。
