# Skill Vetter：第三方代码安全审查

在安装或运行任何来源不明的代码前，执行六项安全检查并输出报告。

> 背景：2026年2月 ClawHavoc 事件后创建。ClawHub 上约 20% 技能为恶意软件。
> 原则：**先审后装，宁可误报不可漏报。**

---

## 触发时机

以下情况自动触发（不需要用户单独调用）：

- 用户说"安装这个"、"install"、"运行这个脚本"
- 用户问"这个安全吗"、"这个可信吗"
- 用户粘贴了来源不明的代码/脚本/配置
- 用户分享了一个 skill/插件/工具的链接要求安装

也可主动调用：`/skill-vetter`

---

## 执行步骤

### 1. 获取代码

- 如果是文件/目录：读取所有代码文件
- 如果是 URL：fetch 仓库内容
- 如果是粘贴的代码块：直接分析

### 2. 六项安全扫描

逐项检查以下威胁，每项给出 🟢安全 / 🟡可疑 / 🔴危险 评级：

#### 🔴 反弹 Shell / 远程代码执行（RCE）
- `exec()`、`eval()`、`subprocess` 带动态输入
- `os.system()`、`child_process.exec()` 拼接用户输入
- 反弹 shell 模式（`/dev/tcp`、`nc -e`、`bash -i`）
- WebSocket/HTTP 接收并执行远程命令

#### 🔑 凭证 / API Key 窃取
- 读取 `~/.ssh/`、`~/.aws/`、`.env`、`credentials`、`config.json`
- 读取环境变量并外传（`os.environ` + HTTP 请求）
- 搜索文件系统中的 key/token 模式

#### 🌀 代码混淆 / 隐藏 Payload
- Base64 编码的可执行内容
- 字符拼接构造命令（`chr()` 拼接、`\x` 转义）
- 压缩/编码后动态解压执行
- 超长单行、不可读变量名

#### 🌐 可疑网络请求
- 非必要的外部 HTTP/HTTPS 请求
- 数据上传到非官方域名
- DNS 隧道模式
- 未说明用途的 webhook URL

#### 📁 文件系统越权
- 写入 `~/.claude/`、`~/.ssh/`、`~/.bashrc` 等敏感路径
- 递归删除操作（`rm -rf`、`shutil.rmtree`）
- 修改系统配置文件
- 创建隐藏文件/目录

#### ⛓️ 供应链投毒特征
- 依赖中包含拼写近似的包名（typosquatting）
- `postinstall` 脚本执行网络请求
- 锁文件中的哈希与官方不一致
- 从非官方源安装依赖

### 3. 输出报告

使用以下固定格式：

```
## 🛡️ 安全审查报告

**审查对象**：<名称/来源>
**审查时间**：<日期>

| 检查项 | 评级 | 说明 |
|--------|------|------|
| 反弹 Shell / RCE | 🟢/🟡/🔴 | <具体发现> |
| 凭证窃取 | 🟢/🟡/🔴 | <具体发现> |
| 代码混淆 | 🟢/🟡/🔴 | <具体发现> |
| 可疑网络请求 | 🟢/🟡/🔴 | <具体发现> |
| 文件系统越权 | 🟢/🟡/🔴 | <具体发现> |
| 供应链投毒 | 🟢/🟡/🔴 | <具体发现> |

**综合评级**：🟢 安全 / 🟡 需注意 / 🔴 建议不安装

**建议**：<是否安装、需要注意什么、是否需要沙箱运行>
```

### 4. 必要性检查（FP 快筛）

安全扫描通过后，在安装前做一次必要性判断：

1. 列出当前项目已有的 skills（读取 `.claude/skills/` 目录）
2. 对比待安装 skill 的核心功能
3. 回答以下两个问题：
   - **功能重叠**：已有哪个 skill 覆盖了相同或更多的场景？
   - **增量价值**：装了之后能做到现有 skill 做不到的事吗？

在报告末尾追加：

```
## 🔍 必要性检查

| 对比项 | 结论 |
|--------|------|
| 功能重叠 | <与哪个现有 skill 重叠，或"无重叠"> |
| 增量价值 | ✅ 有 / ⚠️ 无（<说明>） |

**建议**：安装 / 跳过（已有 <skill名> 覆盖）
```

- 有增量价值 → 继续走安装流程
- 无增量价值 → **建议跳过，告知用户已有哪个 skill 覆盖，等用户确认后再决定**

### 5. 根据评级行动

- **全 🟢 且有增量价值**：告知用户安全，可以安装
- **全 🟢 但无增量价值**：建议跳过，等用户确认
- **有 🟡**：列出可疑点，让用户决定是否继续
- **有 🔴**：**明确建议不安装**，解释具体风险，不要帮用户执行安装

---

## 注意

- 宁可误报不可漏报——可疑就标 🟡
- 不要因为代码"看起来正常"就跳过检查，混淆代码就是设计来看起来正常的
- 即使用户坚持要装，🔴 评级时仍要再次提醒风险
- 审查完成前不执行任何安装操作
