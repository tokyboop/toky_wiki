# FP：第一性原理自检

当用户说"fp"，立即停下来，做一次从根本出发的自我审视。

> 核心精神：**剥掉所有"因为一直这样做"的惯性，回到"这件事的本质是什么"。**

---

## 触发时机

- 用户说 `fp`、`/fp`
- 用户说"回到第一性原理"、"从头想想"、"重新审视"

---

## 执行步骤

### 1. 暂停，回顾当前状态

先用一句话总结：**我现在在做什么，试图达成什么目标。**

### 2. 五问自检

按顺序回答以下五个问题，每个用 1-2 句话，直击要害：

#### Q1：问题定义对吗？
> 我在解决的问题，真的是用户需要解决的问题吗？还是我在解决一个衍生问题、表面症状、或自己制造的问题？

#### Q2：有哪些隐含假设？
> 我的方案建立在哪些没有验证的假设上？这些假设成立吗？列出来。

#### Q3：有没有更简单的路径？
> 如果从零开始，只看目标和约束，最简单的解法是什么？当前方案是不是过度工程了？

#### Q4：方向有没有漂移？
> 我现在做的事，还在用户最初的需求范围内吗？有没有不知不觉跑偏了？

#### Q5：卡住的真正原因是什么？
> 如果当前遇到阻碍，根本原因是什么？是技术限制、信息不足、还是方向本身有问题？

### 3. 输出结论

用以下格式输出：

```
## FP 自检

**当前在做**：<一句话>

| 自检项 | 结论 |
|--------|------|
| 问题定义 | ✅ 正确 / ⚠️ <偏差说明> |
| 隐含假设 | <列出关键假设及是否成立> |
| 更简路径 | ✅ 已是最简 / 💡 <更简方案> |
| 方向漂移 | ✅ 未漂移 / ⚠️ <漂移说明> |
| 卡点根因 | ✅ 无卡点 / 🔍 <根因> |

**行动**：<继续当前方向 / 调整方向 / 建议与用户确认>
```

### 4. 根据结论行动

- **全 ✅**：继续当前方向，告诉用户"自检通过，方向没问题"
- **有 ⚠️ 或 💡**：暂停当前工作，把发现告诉用户，等用户决定是否调整
- **发现严重偏差**：明确建议回退，说清楚应该回到哪一步

---

## 注意

- FP 的价值在于**诚实**——不要为了好看全打 ✅，真有问题就说
- 重点是**发现盲点**，不是走过场
- 自检要快，不要写长文，每条 1-2 句话
- 用户可能在你跑偏时才说 fp，所以要特别认真对待方向漂移和问题定义
