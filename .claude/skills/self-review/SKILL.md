# Skill: self-review

## 触发方式
```
/self-review <任务描述> [rounds=3]
```

## 用途
对任意 AI 输出做纵切式自我改进循环，提升生成质量。

## 执行步骤

### Step 1：前置 gate
根据任务类型，推断要检查的维度列表（纵切，每轮一个）。

任务类型 → 维度顺序：
- **SQL**：NULL处理 → 时区/日期 → 性能（全表扫描）→ 业务定义符合度
- **运营报告**：数字准确性 → 结论有无支撑 → 建议可操作性 → 异常遗漏
- **Prompt**：角色定位 → 输出格式 → 歧义表达 → 收敛条件

### Step 2：执行任务
正常完成用户要求的任务，输出初版结果。

### Step 3：纵切 self-review（最多 rounds 轮）

每轮使用以下 prompt 结构：
```
你是一个严格的 [任务类型] reviewer。
你刚才输出了以下内容：
---
[上一轮结果]
---
本轮只检查以下单一维度：[当前维度]
判断是否有问题（✅ 或 ❌ + 一句理由）。

如果有 ❌，直接输出完整修改版本。
如果 ✅，输出 LGTM + 置信度（0-100）。
```

### Step 4：后置 gate
- 置信度 < 80 → 强制再跑一轮（不信模型自评）
- 达到轮次上限 → 停止，输出当前最佳版本
- 输出变更日志（每轮改了什么）

## 输出格式
```
[最终结果]

---
Review 日志：
- Round 1（维度：XX）：✅/❌ → [改了什么]
- Round 2（维度：XX）：✅/❌ → [改了什么]
- 收敛于第 N 轮，置信度 XX
```
