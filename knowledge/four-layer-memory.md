# 四层记忆系统 — 架构蒸馏

> 蒸馏日期：2026-03-21
> 置信度：C1（自建系统，经验证）

---

## 根本问题

AI没有持久记忆，每次对话从零开始。制作人的原则、偏好、文章、项目决策散落各处，无法系统性地影响AI行为。
需要一个**分层记忆系统**，让不同权重、不同范围的知识各归其位。

---

## 核心框架：四层记忆

```
L0 绝对原则（8条）     不可违反，跨所有项目，修改需制作人确认
L1 跨项目记忆          个人层，所有项目通用
  ├ L1A 偏好           非强制参考（如"减少打断优先于增加入口"）
  ├ L1B 反模式（3条）  审核时必查（认知陷阱/设计反模式/旧框架套新事物）
  ├ L1C 我的文章（15篇）最高权重=C1，与L1D冲突时优先
  └ L1D 外部知识       带置信度标记，需经审阅晋升
L2 项目记忆            本项目定点/决策/资产
L3 工作记忆            当前会话状态，跨会话持续
```

### 关键设计决策

| 决策 | 理由 |
|---|---|
| L0独立于L1 | 原则是"法律"，偏好是"习惯"，不可混淆 |
| L1C > L1D | 自己的方法论权重高于外部知识，防止AI被外部观点带偏 |
| 置信度分5级 | 防止未验证知识被当成事实用于决策 |
| 只有制作人能晋升C4→C2 | AI不可自行提升外部知识的权威性 |

---

## 置信度体系

| 等级 | 含义 | 能否用于决策 |
|---|---|---|
| C1 经验证 | 实践证明有效 | 直接用 |
| C2 已采纳 | 制作人认可 | 可以用，标来源 |
| C3 有参考 | 有一定价值 | 仅作"另一种思路" |
| C4 待审阅 | 刚进来 | 不可用于关键决策 |
| C5 线索 | 未消化 | 不可引用 |

**晋升路径**：C5→AI消化→C4→**制作人审阅**→C2/C3→实践验证→C1

---

## L0 八条绝对原则

1. 意图树方法
2. 广深全思维框架（广=穷举，深=追问本质，全=全局影响）
3. 理性优先于感性（量化打分，有罪推定）
4. 立项流程（原则→定点→发散→筛选→反刍）
5. 长线设计哲学（有限资源→无限可玩性）
6. 工程三问与债务管理（目的/拓展/稳定）
7. 先判断目标成立性
8. 透过现象看本质（先"为什么"再"怎么做"）

## 和我的关系

这是我的AI记忆架构。核心洞察：
- 记忆必须分层分权重，不是所有知识平等
- 自己的文章（L1C）权重最高，这是"AI内化制作人思维"的关键机制
- 置信度体系防止AI把未验证的外部知识当真理用
- 人控制晋升门槛，AI控制检索效率，各司其职
