# Claude Code Auto Dream：后台记忆整理机制

> 来源：卡兹克 2026-04-03（灰度期间实测）
> 关联：`claude-code-memory-architecture.md`（Memory 类型体系）

---

## 什么是 Auto Dream

Claude Code 的后台记忆巩固功能。在你打开新对话干活时，系统在后台偷偷跑一个子 Agent，对 Auto Memory 积累的笔记做清洗整理。

**核心价值**：解决 Auto Memory 长期使用后的"噪音库"问题——矛盾记录并存、相对日期失效、过期信息堆积。

---

## 触发条件

自动触发需同时满足：
1. 距上次整理 **≥ 24 小时**
2. 中间积累了 **≥ 5 个对话记录**

手动触发：`/dream`（灰度期间不稳定，可能报 `Unknown skill: dream`）

开关位置：`/memory` → `auto-dream` 选项

---

## 整理流程（4 步）

```
1. Orient（定向）
   └── 读取整个记忆目录，搞清楚都记了什么、文件关系

2. 信号搜集
   └── 翻历史对话 JSONL 日志，重点捞：
       · 你纠正它的地方
       · 你让它记住的东西
       · 反复出现的主题
       · 重要决策

3. 巩固（核心）
   └── 信号 vs 现有记忆逐一比对：
       · 合并重复信息
       · 解决矛盾记录
       · 相对日期 → 真实日期
       · 删除过期/无效信息

4. 修剪 & 索引
   └── 清理冗余，更新 MEMORY.md 索引
```

---

## 四层记忆架构（完整版）

| 层级 | 名称 | 作用 | 特点 |
|------|------|------|------|
| L1 | CLAUDE.md | 你主动写的规则/规范 | 最高权限，手动维护 |
| L2 | Auto Memory | Claude 工作中自动记的笔记 | 随手记，会随时间劣化 |
| L3 | Session Memory | 当前对话上下文 | 对话结束即消失，原始 JSONL 留本地 |
| L4 | Auto Dream | 后台巩固层 | 定期清理优化 L2，不触碰源代码 |

**L3 的关键**：Session Memory 对话结束后消失，但 JSONL 日志留在本地，Auto Dream 做梦时会来翻这些日志捞有价值的信息。

---

## 安全边界

- 做梦期间项目代码**只读**，只能写记忆目录
- 同一项目同时只有一个 Auto Dream 在跑
- 原始对话 JSONL 不会被删除（"选择性遗忘"：记忆文件清了，但旧账随时能翻）

## 常见问题

**Auto Memory 没开 Auto Dream 会怎样？**
记忆文件只会增不会减，用久了矛盾、过期信息堆积，反而让 Claude 越用越蠢。

**做梦完成后能看到报告吗？**
自动触发时看不到；手动 `/dream` 触发（需有权限）会输出一个整理结果摘要。
