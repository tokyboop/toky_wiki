# AI正在吞噬所有软件：五个论断

> 来源：卡兹克公众号（2026年）
> 视角：10年用户体验设计师 + 30人公司创始人
> 参照：Marc Andreessen 2011年《Why Software Is Eating the World》

---

## 背景

2011年，Andreessen说"Software is eating the world"，软件吞噬世界，成为硅谷信仰。
15年后，软件确实吞噬了世界。
而现在：**Software was eaten by AI**。主语从主动吞噬的software，变成了被动遭吞噬的software。

---

## 论断一：人人都可以造软件了

Vibe coding：你不需要会Python、不需要懂React、不需要知道API是什么，只需要打开Claude Code或Codex，用人话描述需求，AI帮你写代码并运行。

**影响：**
- 软件开发从"需要多年学习的专业技能"→"像用Excel一样的基础能力"
- 软件生产成本：几十万 → 接近零
- 大神还是大神，但开发软件这件事被AI集体平权了

这只是故事的开头。

---

## 论断二：软件从资产变成耗材

**SaaS的核心前提是：软件很难做。**

过去的商业逻辑：
- 花几千万雇工程师做软件
- 软件难以复制（代码量大、架构复杂、经验积累深）
- 用户一旦开始用，数据沉淀、流程跑起来，迁移成本极高
- 靠订阅收费，持续印钞

现在这个前提开始动摇：
- 一小时内用Claude Code搓出来的图片处理工具，能满足大多数人大多数需求
- "日抛软件"：有新需求就重新做一个，比在旧软件上迭代还快
- 耗材不需要维护，不需要订阅

**资产 vs 耗材的区别：**
- 资产：投入重金打造，长期持有，不断增值
- 耗材：用完就扔，随时替换

> SaaS股票观察：不少公司NDR（净收入留存率）从高位回落，拉新越来越贵，老客户扩张放缓。

软件，正在变得不再是资产。

---

## 论断三：Agent填平了人机交互的鸿沟

**软件的本质是什么？** 是一个翻译层。

人想让计算机帮忙做事，但计算机听不懂人话，所以需要UI作为中间物来沟通。所有的button、菜单、输入框、下拉列表……存在的原因都是人和机器之间有语言鸿沟。

**交互设计领域的核心命题（About Face 4）：**
- 实现模型：系统实际运作的方式
- 心理模型：用户以为它运作的方式
- 设计师的价值：弥合两者之间的差距

**但如果Agent填平了这道鸿沟呢？**

以报销为例：
- 以前：30步操作（选类目 → 填字段 → 上传发票 → 选审批人 → 提交），用户的真实决策只有一个——"我出差花了这些钱"
- 现在：对Agent说"上周出差北京三天，帮我把报销提了"，Agent自动读行程、匹配发票、填表单、选审批人、提交

**产品形态的演变：**
- 过去：App、网站、小程序（有界面，给人用）
- 未来：**Skills**（无界面，给Agent调用）

> 当Agent填平人机交互的鸿沟，大量界面根本不需要存在了。机票查询、会议室预订、数据报表——Agent直接调Skill。

**企业核心能力正在切换：**
- 过去：做出界面直观、交互流畅、体验舒服的好产品
- 未来：把自己的业务封装成可被Agent调用的Skill

---

## 论断四：你的用户可能不再是人了

**过去所有软件设计都建立在一个隐含假设：用户是人类。**

用户画像、用户体验地图、可用性测试、眼动测试、NPS、A/B Test、漏斗分析……整套方法论都围绕人来的。

**Agent时代，这个前提开始动摇：**

场景：你的Agent知道你下周三有上海的会议，它自己判断你需要机票，自己查航班、比价、下单，完成后通知你"东航14:20，靠窗，已订"。

全程没有任何人类操作。从需求产生到任务完成，经过多个Skill，用户是Agent，不是人。

**产品优化对象的切换：**
| 过去 | 未来 |
|------|------|
| 按钮放哪、字多大 | 接口是否稳定 |
| 流程顺不顺 | 权限体系是否标准化 |
| 首屏转化率 | 调用成本是否可预测 |
| 让人用得爽 | 让Agent调得爽 |
| 争人的时间 | 争Agent的调用权 |

**新的增长指标：**
- 被多少主流Agent默认接入
- 被多少工作流引用
- 在多少自动化链条里占据稳定节点

产品的脸（UI）变得没那么重要，产品的骨架（API文档、权限说明、调用协议）变得前所未有地重要。

---

## 论断五：中间层的消亡

**技术变革史上的规律：** 每一次重大变革，本质都是提高信息流转效率，消灭中间层。
- 印刷术消灭抄写员
- 电话消灭信使
- 互联网消灭信息中间商
- 电商消灭经销商

**软件本身就是中间层。** Andreessen说软件吞噬世界时，这个中间层在扩张。现在AI来吞噬软件，这个中间层本身正在被消灭。

整个软件从厚重的产品，被拆解成轻量的能力原子。

**类比：电的普及**
- 电网前：工厂自己买发电机、自己发电、自己维护
- 电网后：插上插头，电就来了

AI + Agent + Skill = 新的电网，软件 = 正在被淘汰的发电机。

**组织结构同样在被重塑：**

公司的管理层本质也是中间层——把CEO的意图翻译成下面能执行的指令，再把下面的进展翻译成上面能理解的汇报。管理者 = 组织里的UI。

当业务经验、数据都被封装成Skills后：
- "帮我看过去三个月各业务线利润趋势" → Agent调Skill，10分钟出报告（以前要两三天）
- "帮我把OKR分解到各小组" → Agent了解战略和各组能力边界，先出分解方案，人做最终判断

**不会消失的管理角色：** 需要战略判断、处理人际关系、在不确定中拍板的
**会被压缩的：** 主要职能是信息搬运和任务分发的中间层角色

---

## 总结框架

```
软件是人与机器之间的中间层
    → Agent填平了这道鸿沟
    → 软件被吞噬

管理层是意图与执行之间的中间层
    → Agent同样在填平这道鸿沟
    → 组织结构被重塑

从产品到公司到行业，中间层无处不在
    → AI正在同时对所有中间层发起冲击
    → 颠覆速度远比15年前快
```

---

## 给从业者的思考

**产品方向：** 不要只问"界面好不好用"，要问"能不能被Agent接入、调用是否稳定、能否被组合进工作流"

**职业方向：** 识别自己的工作中哪些是信息搬运（中间层），哪些是真正的判断和创造（不可替代）

**认知方向：** 柯达不是不知道变化，是一直想着怎么让胶卷更好，而没有想"一个没有胶卷的世界会是什么样"

> "时代不会问你准备好了没有，它只会无情翻到下一页。"
