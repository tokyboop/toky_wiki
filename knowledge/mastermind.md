# 主脑 MasterMind 蒸馏笔记

## 根本问题
AI 工具没有持久记忆——每次开新对话，上下文全丢，人要重复解释背景，累且容易断。

## 核心框架（4层）

```
宪法层  → 最高原则：记什么、不记什么、输出格式是什么
规则层  → CLAUDE.md：遇到XX情况怎么做（行为规范，不是记忆）
执行层  → 19个Skills：自动化的具体动作
数据层  → 按类型分目录存储（tasks/ projects/ inbox/ 等）
```

## 记忆如何流转

```
对话结束 → context-handoff skill → 摘要写入 _index.md
新对话开始 → CLAUDE.md 告诉 Claude → 先读 _index.md → 直接知道上次在哪
```

`_index.md` = 工作记忆（当前焦点 + 待续事项 + 最重要的3件事）

## 后置链（自动化的关键）

任何文件变动后，CLAUDE.md 规则自动触发：
```
文件变动 → sync_index.py → gen_dashboard.py → git push
```
不需要人手动维护一致性。

## CC 用户能复刻多少？

| 能力 | 主脑(OpenCode) | CC用户 |
|------|---------------|--------|
| 持久记忆 | ✅ | ✅ 靠 `_index.md` |
| 后置链自动触发 | ✅ | ✅ 靠 `CLAUDE.md` 规则 |
| 后台 Agent 异步执行 | ✅ API控制 | ❌ 需手动触发 |
| 定时任务 | ✅ | ⚠️ 需配 Windows 计划任务 |

**核心差距**：主脑用 OpenCode API 可以程序化控制每次会话加载什么，CC 不行。
但记忆断裂这个核心问题，CC 用 `_index.md` + `CLAUDE.md` 就能基本解决。

## 跨设备解法

把 `_index.md` 放在 GitHub 同步的仓库里（如 toky_wiki），换设备前 git pull 即可。

## 什么时候该想到它

- 你发现自己每次开新对话都在重复解释背景
- 你想让 AI 记住你的长期目标和偏好
- 你有多个持续进行的项目需要在会话间保持上下文
