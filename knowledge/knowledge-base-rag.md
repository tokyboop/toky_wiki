# 个人知识库（记忆宫殿）蒸馏笔记

## 根本问题
多平台信息超载时，人脑检索失效——你知道"在哪里聊过某件事"，但找不到。
关键词搜索没用，因为你记住的是意思，不是原文。

## 记忆宫殿比喻
> 把散落各处的书收进图书馆，给每本书编目，
> 然后用一个索引系统让你按"意思"而非"书名"找书。

## 核心三步

```
1. 采集  → 打通各平台获取通道（本地文件/API/爬虫）
2. 索引  → 切块 + 打标签 + 向量化 → 存入数据库
3. 检索  → 输入问题 → 语义匹配 → 返回相关片段
```

## 关键概念：向量 vs 关键词

| | 关键词搜索 | 向量搜索（语义搜索）|
|--|--|--|
| 找"苹果很甜" | 必须包含"苹果" | "这个水果含糖量高"也能找到 |
| 优势 | 精确、快 | 能理解意思 |
| 适合 | 文件名、代码 | 对话记录、笔记、知识 |

## 省 Token 策略：增量更新

```
不是每次全部重跑！

数据库加一个字段：processed = true/false
新内容进来 → 只处理 processed=false 的 → 处理完标记 true
```

建库费token，查询极便宜（只有一句问题需要向量化）。

## 两个阶段的成本

```
建索引（一次性）：费token，费时间，但只做一次
查询（每次）：极便宜，只有你的问题需要处理
```

## 现实痛点 & 难度

| 数据源 | 难度 | 方式 |
|--------|------|------|
| 本地文件 | ⭐ 简单 | 直接读路径 |
| 飞书/Notion | ⭐⭐ 中等 | 官方API + token |
| 网页 | ⭐⭐ 中等 | 爬虫 |
| 微信 | ⭐⭐⭐⭐ 很难 | 几乎没有官方API |

## 什么时候该想到它

- 你有大量跨平台的信息（聊天记录、文章、笔记）
- 你经常找不到"之前看过/聊过的某个东西"
- 你想让 AI 基于你的历史知识来回答问题

## 这个东西叫什么

**RAG（Retrieval-Augmented Generation）** = 检索增强生成
先检索相关内容，再让 AI 基于检索结果回答，而不是靠 AI 自己的记忆。
